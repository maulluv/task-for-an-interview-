# Mini Request System

Невеликий веб-додаток — спрощена система подачі та обробки заявок.
Користувач створює заявки, менеджер змінює їхній статус. Усі дані
зберігаються в `localStorage`, тому не зникають після перезавантаження.

## 🔗 Демо

- **Live:** https://mini-request-system-one.vercel.app
- **Репозиторій:** https://github.com/maulluv/task-for-an-interview-

## 🛠️ Стек

- **React 19** + **Vite**
- **Zustand** — управління станом (+ `persist` для localStorage)
- **Tailwind CSS 4** + **DaisyUI** — стилі
- **date-fns** — форматування дат
- **ESLint** — лінтинг

## ✨ Функціонал

Додаток має дві ролі, перемикач — у шапці.

### User
- Створення заявки (Title + Description) з валідацією
- Список власних заявок (назва, опис, статус, дата)
- Редагування заявки (лише зі статусом `new`)
- Видалення заявки
- Сортування за датою (нові / старі зверху)

### Manager
- Список усіх заявок (з ID)
- Зміна статусу: `new → in progress → done`
- Фільтр за статусом (всі / new / in progress / done)
- Сортування за датою

## 🚀 Запуск

Проєкт лежить у папці `front`.

```bash
cd front
npm install      # встановити залежності
npm run dev      # запустити в режимі розробки
```

Відкрити: http://localhost:5173

### Інші команди

```bash
npm run build    # зібрати продакшн-білд
npm run preview  # переглянути продакшн-білд локально
npm run lint     # перевірити код лінтером
npm run lint:fix # автоматично виправити стиль
```

## 📁 Структура

```
front/src/
├── store/        # Zustand-стор (стан + localStorage)
├── constants/    # статуси, опції фільтра та сортування
├── ui/           # примітиви (Button, Input, Select, Badge…)
├── components/   # спільні компоненти (RoleSwitcher)
├── features/     # доменна логіка заявок (Form, Card, List)
├── pages/        # екрани за роллю (UserView, ManagerView)
└── utils/        # допоміжні функції (форматування дат, сортування)
```

## 💡 Деталі реалізації

- **Стан** централізовано в Zustand-сторі; компоненти підписуються
  лише на потрібні шматки даних.
- **Збереження** через мідлвару `persist` — синхронізація з localStorage
  відбувається автоматично.
- **Фільтр і сортування** — локальний стан екрана (не глобальний),
  бо це UI-стан, а не дані.
- **Переходи статусів** описані мапою `NEXT_STATUS`, тому порушити
  ланцюжок (наприклад, перескочити з `new` одразу в `done`) неможливо.
